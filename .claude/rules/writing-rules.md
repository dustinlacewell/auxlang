# Writing Rules Files

Rules appear in context **every message**. Keep them terse.

## Guidelines

- **Max ~40 lines** per rules file
- **Distill, don't document** - quick reference, not tutorial
- **Point to source** - "see `path/to/example.ts`" instead of inline code walls
- **Essential patterns only** - the gotchas and non-obvious stuff
- **No fluff** - skip motivation, history, alternatives

## Anti-patterns

- Long code examples (point to real files instead)
- Explaining "why" at length
- Step-by-step tutorials
- Duplicating what's obvious from the code
