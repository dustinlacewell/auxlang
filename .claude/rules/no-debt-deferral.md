# No Debt Deferral

When you notice technical debt or duplication, fix it immediately.

## The Rule

If you find yourself thinking:
- "I should extract this, but for now..."
- "Ideally this would be shared, but..."
- "The right fix is X, but Y is faster..."

**Stop.** Do X now. Ask the user if you're unsure whether the refactor is worth it.

## Why

- Deferred debt compounds
- "For now" becomes forever
- You have the context now; you won't later
- The user deserves clean code, not patches

## Practice

1. Notice the debt
2. Extract/refactor immediately
3. If uncertain, ask: "Should I extract this shared logic now?"
